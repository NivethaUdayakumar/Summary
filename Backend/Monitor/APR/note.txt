Use a stable run_key like this:

run_key = f"{project_code}|{rundir}|{stage}"

When the same run is processed again, delete old rows for that run_key from both tables, then insert fresh rows.

1. Final schema
CREATE TABLE IF NOT EXISTS APR_timing_detail (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    run_key TEXT NOT NULL,
    project_code TEXT NOT NULL,
    rundir TEXT NOT NULL,
    stage TEXT NOT NULL,
    mode TEXT,
    check_name TEXT,
    corner TEXT,
    voltage TEXT,
    pathgroup TEXT,
    slack REAL,
    endpoint TEXT,
    startpoint TEXT,
    timing TEXT,
    report TEXT,
    created_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS APR_timing_summary (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    run_key TEXT NOT NULL,
    project_code TEXT NOT NULL,
    rundir TEXT NOT NULL,
    stage TEXT NOT NULL,
    mode TEXT,
    check_name TEXT,
    corner TEXT,
    voltage TEXT,
    pathgroup TEXT,
    wns REAL,
    tns REAL,
    nvp INTEGER,
    created_at TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_apr_timing_detail_run_key
ON APR_timing_detail(run_key);

CREATE INDEX IF NOT EXISTS idx_apr_timing_summary_run_key
ON APR_timing_summary(run_key);
2. How overwrite works

For every run:

DELETE FROM APR_timing_detail WHERE run_key = ?;
DELETE FROM APR_timing_summary WHERE run_key = ?;

Then insert the new rows.
4. What each table contains

APR_timing_detail

Stores every parsed timing path row.

Example:

run_key                    project_code   rundir       stage   mode   check_name   corner   voltage   pathgroup   slack    endpoint   startpoint   timing
P1|/runs/R1|place          P1             /runs/R1     place   FUNC   SETUP        TT       0.80      all         -0.023   U1/Q       U2/D         VIOLATED
P1|/runs/R1|place          P1             /runs/R1     place   FUNC   HOLD         TT       0.80      clk         0.011    U3/Q       U4/D         MET

APR_timing_summary

Stores grouped summary rows.

Example:

run_key                    mode   check_name   corner   voltage   pathgroup   wns      tns      nvp
P1|/runs/R1|place          FUNC   SETUP        TT       0.80      all         -0.023   -1.234   45
P1|/runs/R1|place          FUNC   HOLD         TT       0.80      clk         0.000    0.000    0
5. Query examples

Get all detail rows for one run:

SELECT *
FROM APR_timing_detail
WHERE run_key = ?;

Get all summary rows for one run:

SELECT *
FROM APR_timing_summary
WHERE run_key = ?;

Delete one run before overwrite:

DELETE FROM APR_timing_detail
WHERE run_key = ?;

DELETE FROM APR_timing_summary
WHERE run_key = ?;

7. Important note

Because run_key uses absolute rundir, the same logical run will overwrite only when these three are the same:

project_code
absolute rundir
stage
